# 🛠️ 작업지시서 — 성장(RPG) 30레벨 → 100레벨 확장

> 대상 AI(코덱스)에게 그대로 전달하는 지시서. 한국어로 읽되 코드 식별자는 영어 그대로.

## 0. 한 줄 요약
현재 캐릭터 강화·아이템 해금이 **Lv30 부근에서 멈춤**. 이를 **Lv100까지** 끊김 없이 확장한다.
**새 이미지 파일 없이도 즉시 작동**(SVG/이모지/CSS 폴백)해야 하고, 고레벨의 화려함은 **오라(그림 불필요)** 로 준다. 결과는 **ZIP**으로 제출.

## 1. 대상 파일 / 절대 규칙
- 수정 파일: **`sinli2655/index.html` 단 하나** (모든 로직·상수가 이 파일 안에 있음). 새 스크립트 파일로 분리 금지.
- Firebase 데이터 구조와 **100% 호환**: 학생은 `rooms/<반>/rpg/students/<id> = {xp, stats:{int,wis,str,dex,cha,mnd}, job, head, pet, frame, warning, pw}`.
  → **기존 아이템 id·레벨·필드 변경 금지. "추가"만 한다.** (학생 데이터 마이그레이션 불필요해야 함)
- 기존 기능(상벌점 지급, 미션, 자리배치 모드, 학생 로그인, 효과음 등) **회귀 금지**.

## 2. 현재 상태 (수정 지점 — index.html 내)
```
레벨곡선:  const RPG_XP_BASE=50, RPG_XP_STEP=30;  rpgLevel(xp): need(L)=50+(L-1)*30
           (Lv100 누적 XP ≈ 150,480 — 사실상 도달 난망, §3-4 참고)
아바타키:  rpgAvatarKey(job,level,gender): tier = level>=30?3:(>=20?2:1) → "job_g" | "job_g_2" | "job_g_3"
           실제 이미지: 전 직업 base(job_g)만 존재 + knight_m_2/knight_f_2 만 존재.
           나머지 상위 티어는 <img onerror>로 base 폴백.
오라:      const RPG_AURA(직업별 색/기호), rpgAuraTier(level)= >=40?3:>=25?2:>=15?1:0,
           rpgAuraOverlay(level,px,job) — CSS/DOM만으로 글로우+반짝임(그림 없음).
아이템:    RPG_ITEMS_HEAD: flower2,star4,horns6,halo9,crown12  (각 항목에 svg 폴백 + assets/items/head/<id>.png)
           RPG_ITEMS_PET : chick3,cat7,dog11,fox16,unicorn22,dragon30 (emoji 폴백 + png, 알→부화 level>=해금+hatch(기본4))
           RPG_ITEMS_FRAME: wood5,silver10,gold18,rainbow28 (CSS border/box-shadow, 이미지 없음)
해금판정:  rpgPickerRow(...): unlocked = level >= it.level  (교사 모달 + 학생 개인화면 공용)
시각화:    rpgHeadVisual(headId,px)=img(png)+onerror→svg,  rpgPetVisual(petId,px,level)=img(png)+onerror→emoji,
           rpgFrameStyle(frameId)=css
```

## 3. 해야 할 작업

### 3-1. 아이템을 Lv100까지 확장 ⭐(핵심)
세 카탈로그(`RPG_ITEMS_HEAD/PET/FRAME`)에 **고레벨 아이템을 추가**한다(기존 항목은 그대로, 뒤에 이어붙임). 새 항목도 **이미지 없이 즉시 표시**되도록 기존 방식의 폴백을 반드시 포함:
- **HEAD**: 각 항목에 `svg`(간단한 인라인 SVG) 필수. png는 있으면 자동 사용(rpgHeadVisual가 처리).
- **PET**: 각 항목에 `emoji` 필수. png 있으면 자동 사용. (원하면 `hatch`도)
- **FRAME**: 각 항목에 `style`(CSS border/box-shadow) — 이미지 불필요.

권장 배치(레벨은 조정 가능하되 **Lv1~100 내내 해금이 끊기지 않게** 촘촘히):
```
HEAD 추가 예: {마법사모자 15}{나뭇잎관 20}{큐빅별 26}{날개 33}{월계관 42}{불꽃왕관 55}{은하후광 68}{불사조깃털 82}{전설왕관 100}
PET  추가 예: {토끼🐰 26}{펭귄🐧 34}{불사조🔥 42}{그리핀🦅 55}{페가수스🐴 68}{크라켄🐙 82}{신룡🐲 100}
FRAME추가 예: {크리스탈 40}{불꽃 55}{은하 70}{전설 85}{신화 100}
```
(이름·모티브는 자유. 핵심은 개수와 레벨 분포. 각 항목의 `id`는 소문자 영문 신규값.)

### 3-2. 오라 확장 — 이미지 불필요, 고레벨 존재감의 핵심 ⭐
- `rpgAuraTier`를 더 잘게 나눠 **Lv100까지 6단계**로(예: 15/30/45/60/75/90 → tier 1~6).
- `rpgAuraOverlay`가 티어가 오를수록 **점점 더 화려**하게: 글로우 반경·투명도↑, 반짝임 개수·크기↑, 고티어(4~6)에는 **이중 후광 / 회전 링 / 파티클 / 골드·무지개·은하 색** 등 추가. **전부 CSS/DOM 인라인**(외부 파일·라이브러리 금지). 직업별 색은 `RPG_AURA` 유지, 신규 고티어 색상 팔레트 추가 가능.
- CSS 애니메이션 쓰면 `@keyframes`를 `<style>`에 추가(회전/반짝임). 성능 위해 카드 다수에서도 가볍게.

### 3-3. 아바타 상위 티어 "폴백 안전화" (새 그림은 만들지 말 것)
- `rpgAvatarKey` 티어 구간을 Lv100까지 확장(예: 1/20/40/60/80/100 → tier 1~6), 반환은 `job_g` / `job_g_2` … `job_g_6`.
- **중요:** 상위 티어 png가 없을 때 `<img onerror>`가 **한 단계씩 낮은 티어로 순차 폴백**하다가 최종 `job_g` → `novice_g`로 가도록 개선(현재는 곧장 base로). 이러면 이미지 없이도 안 깨지고, 나중에 `knight_m_3.png` 등을 올리면 **자동 적용**.
- 이 변경은 카드(`rpgCardHtml`), 모달(`rpgOpen`), 학생화면(`rpgRenderStudentPage`), 캐릭터뽑기(`actRpgAvatarFrom`) **모든 아바타 출력부**에 일관 적용.
- ⚠️ **새 캐릭터 이미지는 생성하지 않는다**(선생님이 나중에 ChatGPT로 제작). 파일명 규칙만 문서화.

### 3-4. 레벨 곡선 (선택·권장, 기본은 보존)
현재 곡선은 Lv100 누적 ≈15만 XP라 도달 난망. **기본값(50/30)은 유지**하되, 상단에 **대안 곡선을 주석 또는 토글 상수로 제시**(예: 최상위권이 1학년/1년 안에 고레벨을 볼 수 있게 완만화). 학생 `xp` 원값은 그대로, **레벨 환산만** 바뀌므로 데이터는 안 건드림. 실제 채택 여부는 도학원쌤이 결정하게 남겨둘 것.

## 4. 새로 생길 수 있는 에셋 (있으면 화질↑, 없어도 정상 작동) — 파일명 규칙
- 아바타 상위 티어: `assets/avatars/<job>_<m|f>_<2..6>.png`  (job=knight/mage/priest/archer/bard)
- 머리: `assets/items/head/<새id>.png`   / 펫: `assets/items/pet/<새id>.png` (알 `egg.png` 이미 있음)
- 새로 추가한 아이템 `id`·레벨·모티브를 **`NEW_ASSETS.md`** 로 정리(선생님이 나중에 그림 생성할 목록).

## 5. 결과물 — ZIP (`sinli2655_100lv.zip`)
ZIP 안에 다음 포함:
1. **수정된 `index.html`** (이게 핵심 — 클로드가 그대로 검토·배포)
2. **`CHANGES.md`**: 바꾼 상수/함수 목록, 새 아이템 표(id·이름·레벨), 오라 티어 표, (있으면) 대안 레벨곡선, **검증 방법**
3. **`NEW_ASSETS.md`**: 나중에 만들면 좋은(선택) 이미지 파일명 목록

### 검증 방법 (CHANGES.md에 기재 + 코덱스가 직접 확인)
- `index.html`을 브라우저로 열어 **콘솔 에러 0**.
- 임시로 학생 `xp`를 크게(예: Lv40/60/80/100 되게) 넣어:
  - 오라가 티어별로 점점 화려해지는지
  - HEAD/PET/FRAME 아이템이 레벨 구간마다 **끊김 없이 해금**되는지(`rpgPickerRow`에서 🔒→해금)
  - 상위 티어 아바타 이미지가 없어도 **안 깨지고** 하위 티어로 폴백되는지
- **기존 학생**(head/pet/frame 착용, 특정 xp)이 그대로 정상 표시되는지(회귀 없음).
- 교사 카드·모달과 **학생 개인화면 둘 다** 동일하게 보이는지.

## 6. 배경/운영 메모
- 이 앱은 초등 교실용. 단일 `index.html` + Firebase(RTDB) + GitHub Pages(`git push`로 배포).
- **이미지 없이 작동이 최우선**(선생님이 그림은 나중에 추가). 그래서 신규 시각요소는 SVG/이모지/CSS/오라 위주로.
- ZIP 받으면: 도학원쌤/클로드가 `index.html`만 교체 → 검토 → push.

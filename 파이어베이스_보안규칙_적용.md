# 🔐 Firebase 보안 규칙 적용 가이드 (공개 전 마지막 단계)

> 목적: **다른 선생님(예: 와이프)이 써도 서로의 학급 데이터가 안 섞이고, 남이 내 반을 못 보게** 막기.
> 규칙은 **데이터를 지우지 않아요.** 접근 권한만 정해요. 그리고 **언제든 즉시 되돌릴 수 있어요.**

---

## 0. 적용 전 준비 (중요!)
1. 앱을 **본인 계정으로 한 번 열어** 주세요. (열면 자동으로 모든 내 학급에 "소유자 정보"가 채워져요 → 규칙 적용해도 안 잠김)
   - 학급 바에 있는 **반을 하나씩 눌러** 들어가 보고, 아침칠판·성장이 잘 열리는지 확인.
2. **지금 규칙을 백업**해 두기 (되돌릴 때 필요):
   - [Firebase 콘솔](https://console.firebase.google.com/) → 프로젝트 **classscore-sinli**
   - 왼쪽 **Realtime Database → 규칙(Rules)** 탭
   - 현재 규칙 내용을 **복사해서 메모장에 저장** (혹시 되돌릴 때 붙여넣기용)

---

## 1. 규칙 적용
1. Firebase 콘솔 → **Realtime Database → 규칙(Rules)** 탭
2. 기존 내용을 **전부 지우고**, 아래 파일 내용을 **그대로 붙여넣기**
   - 파일: `firebase-rules.json` (이 폴더에 있음)
3. **게시(Publish)** 버튼 클릭

---

## 2. 적용 직후 꼭 테스트 (5분)
아래가 **모두 되면 성공**이에요:

- [ ] 내 계정으로 로그인 → **학급 바에 내 반들이 뜬다**
- [ ] 반을 눌러 들어가서 **아침칠판이 보이고, 글씨를 쓰면 저장된다**
- [ ] **성장** 탭에서 학생 카드가 보이고, **상벌점을 줄 수 있다**
- [ ] **학생 로그인**(QR/링크 `?s=코드`)으로 접속 → 학생이 **자기 번호+비번으로 로그인**되고 **캐릭터 꾸미기(머리·펫·프레임)** 가 저장된다
- [ ] **와이프 계정(다른 구글)** 으로 로그인 → **내 반이 안 보이고**, 자기 반만 새로 만들 수 있다

> ❗ 만약 **어떤 반이 "권한 없음"으로 안 열리면** → 규칙을 **백업본으로 되돌리고**(위 0-2), 저(클로드)에게 **"○○반이 안 열려요"** 라고 알려주세요. 그 반만 소유권을 손봐드릴게요. (데이터는 안전해요.)

---

## 3. 되돌리는 법 (문제 생기면)
- Firebase 콘솔 → Realtime Database → 규칙 → **백업해 둔 예전 규칙을 붙여넣고 게시**
- 끝. 즉시 원상복구돼요. (데이터는 그대로)

---

## 4. 이 규칙이 하는 일 (요약)
| 항목 | 접근 권한 |
|---|---|
| `users/<나>` (내 학급 목록) | **나만** |
| `rooms/<반>` 전체 (성장·자리·과제·점수·칠판…) | 그 반의 **담임/전담(멤버)만** |
| `rooms/<반>/settings…roster`, `rooms/<반>/rpg` | 학생도 읽기 가능(로그인용·캐릭터) |
| 학생의 캐릭터 꾸미기(머리·펫·프레임·직업) | 학생도 쓰기 가능 |
| 학생의 경험치·능력치·점수 | **선생님만** (학생이 못 바꿈) |
| `classCodes`(참여 코드), `studentCodes`(학생 코드) | 로그인/참여에 필요한 만큼만 |

---

## 5. 지금 버전의 한계 (알고 계세요)
- 학생 화면은 아직 **간편 로그인**(번호+비번)이라, 같은 반 학생끼리는 서로 캐릭터를 바꿀 여지가 조금 있어요. (반 내부라 위험도 낮음)
- **정식 공개(모르는 사람도 가입)** 로 넓히려면 나중에 학생 인증 강화 등 한 단계 더 필요해요.
- 지금 목적(**나 + 와이프가 각자 안전하게 사용**)에는 충분해요.

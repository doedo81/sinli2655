# 100레벨 성장(RPG) 시스템 확장 변경사항

## 수정 파일
- `index.html` 하나만 수정했습니다.
- Firebase 저장 구조는 그대로 유지했습니다: `xp`, `stats`, `job`, `head`, `pet`, `frame`, `warning`, `pw`.
- 기존 아이템 id, 레벨, 필드는 변경하지 않고 새 항목만 뒤에 추가했습니다.

## 레벨 / 경험치
- 새 상수: `RPG_MAX_LEVEL = 100`
- 기존 곡선 유지: `RPG_XP_BASE = 50`, `RPG_XP_STEP = 30`
- 대안 곡선 추가: `RPG_LEVEL_CURVE_PRESET = 'gentle'`로 바꾸면 `RPG_XP_GENTLE_BASE = 40`, `RPG_XP_GENTLE_STEP = 18` 사용
- 새 함수:
  - `rpgLevelNeed(lv)`
  - `rpgLevelPct(L)`
- `rpgLevel(xp)`는 100레벨에서 멈추며, 진행 막대는 100%를 넘지 않게 처리했습니다.

## 아바타 단계
- 새 상수/함수:
  - `RPG_AVATAR_TIER_LEVELS`
  - `rpgAvatarTier(level)`
  - `rpgAvatarKeyFromTier(job, genderKey, tier)`
  - `rpgAvatarFallbackKeys(job, level, gender)`
  - `rpgAvatarFallback(img)`
  - `rpgAvatarImg(job, level, gender, heightPx, extraStyle, attrs)`
- 단계 규칙:
  - Lv1: `job_m.png`, `job_f.png`
  - Lv20: `job_m_2.png`, `job_f_2.png`
  - Lv40: `job_m_3.png`, `job_f_3.png`
  - Lv60: `job_m_4.png`, `job_f_4.png`
  - Lv80: `job_m_5.png`, `job_f_5.png`
  - Lv100: `job_m_6.png`, `job_f_6.png`
- 이미지가 없으면 한 단계씩 내려가며 찾습니다. 예: `knight_m_6` → `knight_m_5` → `knight_m_4` → `knight_m_3` → `knight_m_2` → `knight_m` → `novice_m`.
- 적용 위치:
  - 교사용 카드 `rpgCardHtml`
  - 상세 모달 `rpgOpen`
  - 학생 개인 화면 `rpgRenderStudentPage`
  - 활동판 RPG 뽑기 `actRpgAvatarFrom`

## 오라 단계
- `rpgAuraTier(level)`를 6단계로 확장했습니다.

| 레벨 | 티어 | 표시 효과 |
|---:|---:|---|
| 15 | 1 | 기본 글로우 |
| 30 | 2 | 반짝임 증가 |
| 45 | 3 | 회전 링 추가 |
| 60 | 4 | 이중 링/골드 하이라이트 |
| 75 | 5 | 강한 회전 링/입자 증가 |
| 90 | 6 | 신화급 입자/밝은 별 효과 |

- CSS `@keyframes rpgAuraSpin`, `rpgAuraPulse`, `rpgAuraTwinkle`를 추가했습니다.
- 새 이미지나 외부 라이브러리 없이 CSS/DOM 인라인 효과로만 동작합니다.

## 검증 방법
1. `index.html`을 브라우저로 열고 콘솔 오류가 없는지 확인합니다.
2. 임시 학생의 `xp`를 Lv40/Lv60/Lv80/Lv100 수준으로 올려 다음을 확인합니다.
   - 오라가 단계별로 더 화려해지는지
   - HEAD/PET/FRAME 아이템이 레벨 구간마다 해금되는지
   - 없는 상위 아바타 이미지가 하위 단계로 순서대로 fallback 되는지
3. 기존 학생의 `head`, `pet`, `frame`, `job`, `xp` 데이터가 그대로 표시되는지 확인합니다.
4. 교사용 카드, 상세 모달, 학생 개인 화면, 활동판 RPG 뽑기에서 같은 아바타 규칙이 적용되는지 확인합니다.

## 직접 수행한 확인
- `index.html`에서 예전 30레벨 아바타 직접 출력부를 제거하고 `rpgAvatarImg`로 통일했습니다.
- `rpgLevelPct`로 100레벨 진행 막대가 100%를 넘지 않게 했습니다.
- 새 HEAD 항목은 모두 `svg` fallback을 포함합니다.
- 새 PET 항목은 모두 `emoji` fallback과 부화 규칙을 포함합니다.
- 새 FRAME 항목은 모두 CSS `style`만 사용합니다.

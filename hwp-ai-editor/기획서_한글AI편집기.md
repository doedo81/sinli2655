# 한글(HWP/HWPX) AI 편집기 — 기획서

## 1. 배경

기존 `hwpx_edit_v7.py`는 **외부 라이브러리 없이** HWPX(zip+XML)를 안전하게 편집하는 검증된 엔진이다.
서식 XML은 건드리지 않고 텍스트 노드만 교체하므로, 편집 결과를 한/글에서 **원본 서식 그대로** 열 수 있다.
이미 문단·표·셀·글꼴·문단속성·구조검진·공문서원칙까지 25종 기능이 CLI로 구현돼 있다.

그러나 ① CLI라 일반 사용자가 쓰기 어렵고 ② 구형 `.hwp` 미지원 ③ AI 부재 ④ 표 레이아웃을 사람이 손으로 맞춰야 하는 한계가 있다.

## 2. 목표

> **"명령을 대신 실행하는 도구"가 아니라, 양식과 내용을 이해하고 표 레이아웃을 스스로 맞추는 자동 편집기.**

- **웹 앱** (브라우저, 설치 불필요)
- **HWPX + HWP** 둘 다 지원
- **AI 4종**: 자연어 명령 편집 · 공문서 규정 자동 검토 · 초안 자동 생성 · 요약/분석/질의응답

### ★ 최우선 3대 가치
1. **표가 전부다** — 표 정렬·수정·편집·채우기. 높이·좌우 폭 자동 조정, **글자가 긴 셀만 그 칸을 더 넓게**.
2. **표준 양식 준수 자동 편집** — 표준 문서(양식)를 이해하고, **조건에 맞는 부분만** 양식을 지키며 편집.
3. **AI가 자동 계산** — 사람이 행·열·치수를 지정하지 않는다. 폭·높이·글자크기·행 수를 **AI/엔진이 스스로 계산**.

## 3. 아키텍처

```
[브라우저 SPA]  ──HTTPS──▶  [FastAPI 백엔드]  ──▶  [편집 엔진 hwpxlib]
  · 문서/표 뷰어             · 세션·파일 관리         · HwpxDoc  (무손실 편집)
  · AI 채팅 패널             · REST + WebSocket       · AutoTable (표 자동 레이아웃) ★
  · 편집 diff 미리보기       · AI 에이전트 루프        · Template  (양식 스키마 준수) ★
                            │                        · hwp_reader (pyhwp, .hwp 읽기)
                            ▼                        · gov_rules  (공문서 규정)
                       [Claude API] ── tool-use ──▶ 위 엔진 메서드를 "도구"로 호출
                            │
                       [HWP 변환 워커(옵션, Windows)] ── 한/글 COM ── .hwp↔.hwpx 무손실
```

## 4. 핵심 기능

### 4.1 표 자동 레이아웃 (AutoTable) — 구현 착수 완료
`autofit_table(표)` 한 번으로 측정→배분→맞춤을 자동 수행한다.
- **표 전체 폭은 유지**한 채, 열별 최장 내용에 비례해 폭을 재분배 → *긴 내용 열만 넓어진다*.
- **워터필링 배분**: 내용이 작은 열은 제 몫을 온전히 받아 무너지지 않고, 큰 열들이 남은 폭을 나눈다.
- **넘침 처리 우선순위**: ① 열 확장 → ② 행 높이 상향(줄바꿈) → ③ 마지막 수단으로만 그 셀 글자 축소.
- 마무리로 `verify`로 구조(rowCnt/colCnt·병합) 무결성 자동 확인.

### 4.2 표준 양식 준수 (Template) — 구현 완료
- `extract_schema(표, 표번호)`: 헤더행·라벨열·**고정 칸 vs 채움 칸**·열별 스타일·반복 행·헤더 라벨→열 추출.
- `apply_template(표, 표번호, 레코드)`: 채움 칸에만 매핑(고정 칸·서식 불변), 데이터 많으면 `copy_row`로
  행 자동 증설, 채운 뒤 `autofit_table`로 자동 맞춤, 스타일 정합, `validate_template`+`verify` 검증.
- `validate_template`: 고정 칸 불변·글꼴/정렬 일관성·구조·날짜표기 규정 점검.
- CLI: `template-info`, `apply-template --records "값|값;값|값"`.

### 4.3 AI 오케스트레이션 (다음 단계)
Claude tool-use로 저수준 편집 도구 + **고수준 도구**(`autofit_table`, `apply_template`)를 노출.
루프: 현황 조회(`tables`/`paragraphs`) → 편집 → `verify` 자기검증 → 실패 재시도 → 사용자 diff 확정.

## 5. HWP(구형) 전략 — 정직한 3단계

| 단계 | 기능 | 방법 | 완성도 |
|------|------|------|--------|
| 1 (바로) | 분석·요약·규정검토 | `pyhwp`로 .hwp 텍스트/표 추출 → AI 입력 | 읽기 완벽 |
| 2 (편집) | .hwp 편집 | .hwp→.hwpx 변환 후 편집, .hwpx로 저장 | 편집 완벽, 저장은 .hwpx |
| 3 (옵션) | .hwp 무손실 재저장 | Windows 워커 + 한/글 COM으로 .hwpx→.hwp | 완벽(라이선스·인프라 필요) |

순수 리눅스 웹서버에서 구형 `.hwp`를 무손실로 **재저장**하는 무료 경로는 없다.
따라서 편집은 HWPX 네이티브로 수행하고, `.hwp`는 분석/검토(1)와 변환 편집(2)으로 처리한다.

## 6. 로드맵

- **Phase 0 — 엔진 라이브러리화** ✅ : `hwpx_edit_v7.py` → `hwpxlib/`(로직 불변, 웹용 `from_bytes`/`save_bytes` 추가, CLI 하위호환)
- **Phase A — AutoTable** ✅ : `autofit_table` + 회귀 테스트
- **Phase B — Template** ✅ : `extract_schema`·`apply_template`·`validate_template` + 테스트
- **Phase C — AI 오케스트레이션** : Claude tool-use 에이전트
- **Phase D — 웹앱 UI** : 업로드·뷰어·AI 채팅·diff·다운로드
- **Phase E — HWP 지원** : 위 3단계
- **Phase F — 요약·분석·QA + 규정검토·초안생성 완성**

## 7. 검증 기준

- 편집 후 `verify()`가 항상 "이상 없음" (릴리스 게이트).
- AutoTable: 긴 열만 넓어지고 · 겹침 없고 · 표 전체 폭 유지 · 결과를 한/글에서 열어 육안 확인.
- 현재 자동 테스트 `tests/test_engine.py` 5종 통과(왕복 저장, 파싱, 편집+verify, 열폭 확장, 행높이 상향).
